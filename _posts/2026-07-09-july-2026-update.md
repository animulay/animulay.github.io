### C++
- [C++20: An (Almost) Complete Overview](https://www.youtube.com/watch?v=FRkJCvHWdwQ) - [Marc Gregoire](marc.gregoire@nikon.com) - CppCon 2020
  C++20 introduced four major features: modules, ranges, Coroutines and Concepts.<br>
  1. [modules](https://youtu.be/FRkJCvHWdwQ?t=238)<br>
  2. [Ranges](https://youtu.be/FRkJCvHWdwQ?t=545)<br>
      Range
      - is an object referring to a sequence / range of elements<b>
      - is similar to begin/end iterator pair (but does not replace them)<br>
      - allows range adapters to **lazily** transform/filter underlying sequences of elements<br>
      - All standard library algorithms that accept ranges also support<br>
      • Projection (a callback that transforms elements before handing over to the algorithm)<br>
      • Views (transform/filter range: lazily evaluated, non-owning, non-mutating)<br>
      • Range factories (construct views to produce values on demand e.g. sequence of integers)<br>
      • Pipelining (views can be chained using pipes '|' just like the Unix pipe feature)<br>
  4. [Coroutines](https://youtu.be/FRkJCvHWdwQ?t=905)<br>
  5. [Concepts](https://youtu.be/FRkJCvHWdwQ?t=1140)
     - Concepts are a mechanism for specifying requirements / contract for template parameters.<br>
     - The predicates are evaluated at compile time.<br>
     - This feature is more helpful for library developers.<br>


### Database
- [From monolith to Lakebase to LTAP: rethinking the database from storage up](https://www.databricks.com/blog/lakebase-ltap-rethinking-database-storage) by [Reynold Xin](https://www.databricks.com/blog/author/reynold-xin)
- [ODBC Takes an Arrow to the Knee - ADBC](https://www.youtube.com/watch?v=BwTG3ErPDiU) by [Matthew Topol](https://x.com/zeroshade), co-founder @ Columnar.tech, Jan 14, 2026
  - The traditional database drivers (ODBC/JDBC) return data in **row-oriented** format.<br>
  - Using ODBC/JDBC drivers with column-oriented analytical databases result in highly inefficient usage pattern.<br>
  - ADBC is Apache Arrow native. It eliminates the data conversion cost if your application needs to work with column-oriented data.<br>
  - [ADBC Quickstarts](https://github.com/columnar-tech/adbc-quickstarts/) offers sample examples for using ADBC with various databases and query engines.<br>
  - [In-Memory Analytics with Apache Arrow](https://www.goodreads.com/book/show/218647290-in-memory-analytics-with-apache-arrow), 2nd Edition by Matthew Topol<br>


### Infrastructure
- [OSDI '20 - The CacheLib Caching Engine: Design and Experiences at Scale](https://www.youtube.com/watch?v=JIM08lWPvNs)<br>
  - Caching is pervasive, but the systems defer along several axes - performance goals, system topology (in-process vs remote), WL, domain-specific features ...<br>
  - The caches have different goals<br>
  
    | Cache Type | Use-Case | Special Considerations|
    |:-----------|:---------|:----------------------|
    | CDN cache at the edge | static content | prioritize low latency; strict TTL to prevent stale content |
    | Lookaside key-value | session info; user profile | high throughput |
    | SocialGraph | | strict consistency requirements |
    | Storage | user generated content | large capcity |
