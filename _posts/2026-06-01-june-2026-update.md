### Database
- Monster Scale Summit 2025 [Time Travelling at Scale by Richard Hart](https://www.youtube.com/watch?v=URoY4l7f0c8)
  - Richard Hart talks about Pangolin, a purpose-built analytical tree-database created at [Antithesis](https://antithesis.com) to replace [BigQuery](https://cloud.google.com/bigquery).<br>
  - The database is designed to record data (fault injection, logs, code coverage etc.) at every state of the simulation with a performance objective to support 100 concurrent queries with P99 query performance of 5 seconds, over a 1 TB dataset.<br>
  - Implemented in Rust; supports ACID transactions; **append-only** with log-structured merge tree [^1]
  - Uses [Amazon S3](https://aws.amazon.com/s3) for blob storage and [AWS Lambda](https://aws.amazon.com/lambda)
  - Another interesting implementation detail is the use of Skiptree - a data structure inspired by Skip list.


### Data Structure
- [What are skiplists good for?](https://antithesis.com/blog/2026/skiptrees/)<br>
  [Will Wilson](https://antithesis.com/author/will-wilson/) provides the rationale for creating the Skiptree data structure used in the Pangolin analytics tree-database developed in-house.<br>
- **Context**: Antithesis simulation testing results in a branching tree of timelines where each path from root to a leaf node represents a sequence of events leading to a specific outcome (e.g. log message / error / crash). In order to investigate an outcome, users want to know the exact sequence of events leading to that outome, which is essentially traversing up the tree from the leaf node to the root.<br>
- Earlier Antithesis used [BigQuery](https://cloud.google.com/bigquery), but due to this unconventional use-case for an analytics database, every operation resulted in a full table scan. Not only the queries were really slow, but it was also expensive.<br>
- **Solution**: They designed Skiptree - a new tree data structure that's modeled to work like a [Skip list](https://en.wikipedia.org/wiki/Skip_list), where each tree has roughly 50% of the nodes from the levels below.<br>
- Each level of the Skiptree is stored as a separate table in the database and you can get the sequence of events by **JOIN**ing together tables from the leaf node to the root node.<br>

### LLM Engineering
- Webinar recording: [Observing and Improving AI Agents with Langfuse](https://www.youtube.com/watch?v=6A8_aahQ0nE)<br>
  Clemens Rawert, co-founder, Langfuse and Doneyli de Jesus, Solutions Architect, ClickHouse, May 27, 2026<br>
  [Langfuse](https://github.com/langfuse/langfuse) is an open source (MIT license) LLM engineering platform to develop, monitor, evaluate and debug AI applications.<br>
  The case for Langfuse:
  - GenAI is not deterministic
  - problems are hard to anticipate and hard to reproduce - the traditional system metrics look green, but an AI agent may have failed to provide the correct answer ...<br>
    We need both - the traditional observability + AI agent observability<br><br>
    ![Traditional vs AI agent observability](/images/traditional-vs-AI-agent-o11y.png)
  - The solution is an observability loop, not a dashboard.<br><br>
