### C++
- [Effective Ranges: A Tutorial for Using C++2x Ranges - Jeff Garland - CppCon 2023](https://youtu.be/QoaVRQvA6hI)<br>
  • [C++20](https://en.cppreference.com/cpp/20) is the first official standard that introduced [ranges](https://en.cppreference.com/cpp/header/ranges).<br>
  • Ranges are the largest revamp of the Standard Template Library (STL) in 20 years and impacts daily C++ programming dramatically.<br>
  • This tutorial offers an excellent overview of ranges, views and spans.<br> 
  • Ranges algorithms are not fully compatible with STL algorithms.<br>
  • Views are ranges with **lazy** evaluation and lazy evaluation allows for composition of several processing steps. This allows us to write `functional style` code.<br>
  • If you are on older releases of C++, say, [C++14](https://en.cppreference.com/cpp/14) or [C++17](https://en.cppreference.com/cpp/17), then Eric Niebler’s [range-v3](https://github.com/ericniebler/range-v3) library allows you to use ranges.<br>

### GenAI
- [ghost](https://ghost.build/learn)<br>
  • ghost is a hosted database (Postgres) service for agentic AI use-cases<br>
  • emphasizes a Git like fork model to give every agent its own fork (copy) of the production database<br>
  • after testing and validation, changes from the fork are merged into the production database and the fork is discarded<br>
  • enables several agents to work in parallel without affecting the production database<br>
  • offers Skills and MCP integration to work with several frontline products<br>

### Infrastructure
- [Creating our Own Kubernetes & Docker to Run Our Data Infrastructure - Modal](https://www.youtube.com/watch?v=3jJ1GhGkLY0)<br>
- [The architecture of VictoriaLogs](https://docs.google.com/presentation/d/1Qw0V4ssthDwA5HaNjJeJl6JiCkYPzqo6A0y9a9UnXEM)<br>
  by [Aliaksandr Valialkin](https://x.com/valyala), CTO, [VictoriaMetrics](https://victoriametrics.com/)<br>
  • Nice high level overview of various design decisions that make VictoriaLogs really compact, fast and efficient<br>
  • Implemented in Go<br>
  • There are some similarities with [ClickHouse](https://clickhouse.com) e.g.<br>
    - uses LSM tree to take advantage of the fact that the (log) file contents never change<br>
    - logs are split into blocks of upto 2 MiB (compared to 8 MB - the default granule size in ClickHouse)<br>
      Each block fits into typical CPU cache. This cache friendly design decision probably gives it a huge advantage.<br>
    - The sequential disk I/O benefits both the write and read use-cases<br> 
    - column-oriented storage that allows for better data compression ratios<br>
    - use of Bloom Filters<br>
